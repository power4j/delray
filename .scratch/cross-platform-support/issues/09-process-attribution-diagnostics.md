# 09 — 进程归属诊断指标

Status: ready-for-agent

**What to build:** 为进程归属路径增加可测试的诊断数据，量化 lookup 命中、未命中、进程表刷新和未归属原因，帮助区分时序竞态、权限限制和系统边界。

**Blocked by:** 01 — 以 listeners 打通 Linux 进程归属；02 — 保守进程归属规则

## Acceptance Criteria

- [ ] 聚合路径能够记录进程 lookup 命中、未命中和无本机 socket 的计数或字节量。
- [ ] 进程表刷新路径能够记录刷新请求次数、实际刷新次数、刷新成功/失败和刷新耗时。
- [ ] 诊断数据通过内部快照或测试 seam 可观察；本票不要求新增用户可见 UI。
- [ ] 诊断采集不得在聚合线程同步调用 `listeners::get_all()`。
- [ ] 测试覆盖命中、未命中、无 socket、刷新成功、刷新失败和刷新耗时记录。

## Notes

- 本票只建立观测能力，不改变统计归属语义。
- 诊断字段优先保持内部可测试；是否展示到 TUI 或报告另开产品决定。
