# 11 — 短窗口待归属缓冲决策与实现

Status: ready-for-agent

**What to build:** 评估并实现短窗口待归属缓冲：lookup 未命中的近期流量先进入有界 pending 桶，新进程表发布后只在唯一候选成功时归属，否则超时后进入未归属流量。

**Blocked by:** 10 — lookup 未命中时合并刷新进程表

## Acceptance Criteria

- [ ] 先修订规格或 ADR，明确允许短窗口内的历史流量延迟提交或重归属，并说明它与当前「不追溯修改历史流量」语义的关系。
- [ ] `Flow` 或等价内部模型包含足够形成连接键的信息，至少区分本机 IP、本机端口、远端 IP、远端端口和协议。
- [ ] pending 桶有时间上限和容量上限；超时、容量溢出、歧义、查询失败时最终进入未归属流量。
- [ ] pending 提交保持守恒关系：总流量 = 已归属 + pending + 最终未归属。
- [ ] `Last seen` 使用原始流量观测时间，不使用查询完成时间。
- [ ] 测试覆盖持续连接、短连接、端口复用、歧义、刷新失败、pending 超时和容量上限。

## Notes

- 本票是准确率收益最高但语义风险最大的阶段。
- 不直接复制 Sniffnet 的端口级 `get_process_by_port()` 行为；必须保留 Delray 的保守唯一候选规则。
