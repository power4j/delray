# 02 — 保守处理歧义与过期进程查询

**What to build:** 让进程查询在通配监听、共享 socket、身份信息缺失和刷新故障下保持正确；只有唯一候选 PID 才归属，无法可靠确定的流量进入「未归属流量」。

**Blocked by:** 01 — 以 listeners 打通 Linux 进程归属

**Status:** ready-for-agent

- [ ] 具体本机地址优先于同协议、同端口的 `0.0.0.0` 或 `::` 通配监听。
- [ ] 多条候选记录最终对应同一个 PID 时可以正常归属。
- [ ] 多条候选记录对应不同 PID 时返回歧义，不任意选择第一个 PID。
- [ ] 没有候选或候选歧义的流量都计入「未归属流量」。
- [ ] PID 已知但进程名或路径缺失时仍保留进程归属。
- [ ] 一次刷新失败后继续使用最后一次成功的进程表。
- [ ] 旧进程表最多使用 `2 × --proc-refresh`，超过有效期后停止用于新流量归属。
- [ ] 刷新恢复后自动使用新进程表，不追溯修改历史流量。
- [ ] 查询错误写入 stderr 或日志，但不终止抓包、统计或 TUI。
- [ ] 使用可控时间测试有效旧快照、过期旧快照、恢复刷新和错误持续期间的行为。
