# 04 — 输出进程统计身份、路径和 Last seen

**What to build:** 让每个进程统计身份携带稳定可执行文件名、可空路径和最后流量时间，并在 plain 与 JSON/JSONL 中提供一致、可解析的数据。

**Blocked by:** 02 — 保守处理歧义与过期进程查询

**Status:** ready-for-agent

- [ ] 路径可用时以 PID 和可执行路径共同区分进程统计身份。
- [ ] 相同 PID 对应不同路径时创建不同统计身份，不合并历史流量。
- [ ] 路径不可用时按 PID 区分，并保留已知 PID 的进程归属。
- [ ] 进程名缺失时沿用现有未知名称表示，JSON 使用 `name: null`。
- [ ] 每个进程统计身份记录最后一次实际捕获并归属的流量时间。
- [ ] 进程表刷新、名称变化和路径变化不更新 `Last seen`。
- [ ] 未归属流量记录自身的 `Last seen`，零流量仍不创建未归属记录。
- [ ] plain 进程记录增加英文 `Path` 和 `Last Seen`，缺失路径显示 `-`，时间使用 ISO 8601。
- [ ] JSON/JSONL 进程记录增加可空 `path` 和 RFC 3339 `last_seen`；未归属身份使用 `pid: null`、`name: null`、`path: null`。
- [ ] 未归属流量继续正常参与 top-N，不固定置顶或额外增加一行。
- [ ] 测试覆盖身份拆分、双方向流量、时间更新、空值、top-N 和接口流量分区不变量。
