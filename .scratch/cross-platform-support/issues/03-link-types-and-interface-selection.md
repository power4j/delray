# 03 — 支持多种数据链路与编号选网卡

**What to build:** 让 Linux 和 Windows 目标环境中的物理网卡、回环设备和聚合设备能够按实际 pcap 数据链路类型解析，并让长设备名可以通过临时数字编号选择。

**Blocked by:** 01 — 以 listeners 打通 Linux 进程归属

**Status:** ready-for-agent

- [ ] 捕获打开后保存并使用实际 pcap link type，不再固定按 Ethernet 解析。
- [ ] 支持 Ethernet、Raw IPv4、Raw IPv6、NULL、LOOP、Linux SLL 和 Linux SLL2。
- [ ] 固定字节样本覆盖各数据链路类型的 TCP、UDP、IPv4 和 IPv6 解析。
- [ ] 不属于本机的流量继续被忽略，本机方向、本机端口和远端 IP 保持正确。
- [ ] 未支持的数据链路类型返回明确英文错误，不静默产生空统计。
- [ ] 无网卡参数时按 `1, 2, 3...` 显示英文网卡列表、友好描述和完整 pcap 设备名。
- [ ] 位置参数接受当次列表中的数字编号，也继续接受完整设备名。
- [ ] 无效编号和不存在的设备名返回明确英文错误。
- [ ] 数字编号只用于交互选择，不作为跨进程稳定标识；现有完整设备名脚本行为保持可用。
- [ ] Linux 默认路由标记保持尽力而为，非 Linux 平台不依赖该能力。
